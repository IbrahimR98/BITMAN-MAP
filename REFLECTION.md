**Reflection:**



Going into this assignment, I honestly thought it was going to be pretty simple. I figured I could just scrape the BITMAN program pages, pull the course information, and automatically generate a clean visual map. In my head, it sounded straightforward. But once I actually started working on it, I realized that scraping real websites is way messier than it looks.



I tried to automatically pull prerequisite information from the course pages, but the website wasn’t consistent. Some pages were structured differently and the information wasn’t always clearly listed, so even when my code ran, the results weren’t reliable. That part was frustrating.



Eventually, I decided to shift the focus of the project. Instead of trying to fully automate prerequisite extraction which wasn’t reliable, I focused on building a clean 2-year program map that clearly shows core courses, specializations, overlap, flexible learning options, and half-semester courses. I think that was the right decision because the final output is way clearer and more useful.



Using AI helped a lot, especially for my code and debugging errors. It was useful for helping me break the project into smaller steps instead of code, one by on block, instead of one whole script. When I got stuck with errors or weird output, it helped me figure out what might be wrong. That saved me a lot of time. However AI wasn’t perfect. Sometimes it gave code that looked correct but didn’t actually match the structure of the BCIT website. Other times it suggested solutions that were way more complicated than what I needed. I learned pretty quickly that I couldn’t just copy and paste everything and assume it would work. I had to actually test things, read the data myself, and make decisions about what made sense.



Overall, I actually enjoyed the project, even though there were definitely moments where I was confused or annoyed. It showed me that AI is a powerful tool, but it’s not magic. You still need to think critically, understand your data, and be willing to change direction when something isn’t working. I feel a lot more confident now working with larger, more open-ended coding projects than I did before.
